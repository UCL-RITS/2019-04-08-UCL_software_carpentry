  • Command summary so far
  • ls - list files
  • pwd - print working directory
  • man - read the manual of a command
  • cd - change directory
  • Tips:
  • * commands on the command line are usually made up of commands, options and arguments, separated by spaces: "ls -l -h /". "ls" is the command, "-l -h" are options, "/" is th argument.  Some commands may not take arguments (like "pwd"), others can optionally accept an argument (like "ls", which uses as default argument the current directory, if no argument is provided) options are... well... optional. Options can be toghether: `ls -l -h` is the same than `ls -lh`
  • * type "q" to quit the manual of commands
  • cd .. - move up one directory 
  • cd - moves you back to your home directory (similarly: `cd ~`)
  • mkdir - makes a directory i.e. mkdir thesis 
      • then click on ls to see whether the command worked 
      • but writing `mkdir new directory` will create two directories `new` and `directory` so instead of space its best if you use numbers, dashes etc. Or use quotes: `mkdir "new directory"`; however spaces on directories and files are not good practice as they are not understood in the same way by all the operative systems.
    nano - opens the nano text editor
            write something then click `control` `X` to exit 
    nano draft.txt - creates a text file with nano named draft.txt 
    
    
    touch my_file.txt - creates an empty file with the name that you give it 
    ls -l - (l = lower case L) shows information about the files (permissions, owners, date modified)
    
    
    mv thesis/draft.txt thesis/quotes.txt - move or rename file 
    mv -i -is a layer of protection that notifies you if you are about to overwrite a file  (`-i` for interactive)
    
    cp - copy a file to another folder
    cp -r thesis thesis_backup - to copy a folder/ directory  (`-r` as recursive, the directory and the files within in)
    
    rm quotes.txt -  delete/remove file 
    rm - i quotes.txt - deletes a file with a protection layer that notifies you when you are about to delete a file 
    
    ls *.pdb - list all the files that end with .pdb, star represents as many characters as there might be
    ls p*.pdb - list all the files that start with p and end with .pdb (including no characters: i.e., p.pdb)
    ls ?ethane.pdb - if I'm not sure what's the first letter of the name of a file I can replace it with a question mark, ? represents one character 
    ls ???ane.pdb - it looks for any file with three letters before ane.pdb
    ls *ane.pdb - lokks for as any file with as many letters before ane.pdb
    ls [em]*.pdb - return anything with an m or e
    
    
    wc octane.pdb - count the number of words in a file named octane.pdb
    wc -1 *pbd  > lengths.txt - create a lengths.txt file and redirects the results from the wc command within that file 
    sort -n lengths.txt - sort the numbers in the file lengths.txt
    sort -n lengths.txt  > sorted_lengths.txt - sort the numbers in the file lengths.txt and save the results in a new file sorted_lengths.txt
    head -n 1 sorted_length.txt - return the first line of the file sorted_lenghth.txt 
    tail -n 1 sorted length.txt - return the last line of the file  sorted_lenghth.txt 
    wc -1 *pdb | sort -n  | head -n 1 - count the words in any file that ends with pdb, sort them by number and return the first line of the output 
    
    history - shows the commands you've used so far 
    !154 - runs command in line 154 of the history 
    history | tail -n 10 - shows the last 10 commands you've used so far  (or history -10)
